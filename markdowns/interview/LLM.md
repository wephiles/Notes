# AI 大模型面试题 50 个（含精炼答案）

## 基础与总体框架

1. 问：什么是“基础模型/大语言模型（LLM）”？
   答：在海量通用语料上以自回归或自编码目标预训练的通用模型，能迁移到众多下游任务；常见如 Transformer 解码器结构，靠“提示/微调/检索”适配场景。
2. 问：自回归 vs 自编码的区别？
   答：自回归（GPT 类）按从左到右预测下一个 token；自编码（BERT 类）通过掩码重建被遮蔽的 token，更擅长理解；指令对话一般采用自回归。
3. 问：注意力机制的核心公式？
   答：Attention(Q,K,V)=softmax(QK⊤/dk)V\mathrm{Attention}(Q,K,V)=\mathrm{softmax}(QK^\top/\sqrt{d_k})V。通过 Q 与 K 的相似度对 V 加权，捕获长依赖。
4. 问：Transformer 为什么比 RNN 更适合长程依赖？
   答：并行计算、全局依赖建模、梯度传播更稳定；不过原始注意力是 O(n2)O(n^2) 时空复杂度。
5. 问：预训练、SFT、RLHF/RLAIF 各做什么？
   答：预训练学通用知识；SFT（监督微调）学指令跟随与格式；RLHF/RLAIF 通过人类/AI 反馈优化偏好与安全。
6. 问：指令跟随模型与通用 LLM 的差异？
   答：指令模型在“(指令, 回复)”数据上做 SFT（和可能的偏好对齐），在对话/工具调用等任务上更稳。
7. 问：大模型“幻觉”的根因？
   答：语言建模目标追求似然最大化而非事实性；缺少检索/校验；数据偏差与对齐不足。
8. 问：减少幻觉的常见手段？
   答：RAG 检索证据、工具调用校验、依据来源回答、置信度校准、对齐训练（拒答/澄清）、结构化输出约束。

## 数据与分词

1. 问：BPE、SentencePiece、WordPiece 的共性？
   答：都是子词分词法，兼顾词汇覆盖与 OOV；中文常使用 SentencePiece（Unigram/BPE）避免过度拆分。
2. 问：中文分词常见陷阱？
   答：纯 BPE 若基于英文语料，中文可能被切成单字甚至字节；需使用面向中英混合的 tokenizer 或增量训练。
3. 问：Token 与字符/字节的关系？
   答：Token 是统计单元，不等于字符；同一句子在不同 tokenizer 下 token 数差异巨大，影响上下文长度与成本。
4. 问：预训练数据清洗的关键？
   答：去重（近/完全重复）、过滤低质/敏感/垃圾内容、语言/领域比例平衡、元数据保留（来源、时间）便于后续采样。

## 架构与位置编码

1. 问：Multi-Head Attention 的作用？
   答：用多组投影（头）并行关注不同子空间与关系，提升表达力与稳定性。
2. 问：RoPE（旋转位置编码）要点？
   答：将位置信息通过复数旋转注入 Q/K，天然支持长度外推；可用缩放策略提升超长上下文表现。
3. 问：ALiBi 的思想？
   答：对注意力引入与距离成比例的偏置，鼓励邻近关注并具备长度外推能力，参数更少。
4. 问：FlashAttention 的本质？
   答：IO 感知的块状算法，避免显式构造 n×nn\times n 注意力矩阵，降低 HBM 访问与显存占用，显著提速。
5. 问：Transformer 解码器常用正则化？
   答：RMSNorm/LayerNorm、Dropout、残差、激活函数（SwiGLU/GELU），配合学习率计划与权重衰减稳定训练。
6. 问：并行范式：数据/张量/流水线并行的区别？
   答：数据并行复制模型、切批；张量并行切分层内矩阵；流水线并行切分层、跨设备串接。大规模训练常混合使用+FSDP/ZeRO。

## 训练与优化

1. 问：混合精度 BF16 vs FP16？
   答：BF16 指数位更大，数值更稳；在 Ampere+ 通常优于 FP16。权重、激活 BF16，优化器状态可 FP32。
2. 问：学习率与预热策略？
   答：常用线性 warmup（1–3% 步数）+ 余弦/多段下降；过大学习率易发散，过小收敛慢/欠拟合。
3. 问：梯度裁剪与梯度检查点的作用？
   答：裁剪抑制梯度爆炸；检查点减少激活显存（重算前向），以时间换空间。
4. 问：AdamW 为何加 “W”？
   答：将权重衰减与 L2 正则分离并解耦，更稳定；基础优化器仍是 Adam 的一阶与二阶动量。
5. 问：训练内存的粗估公式？
   答：Adam + AMP 约 12–16 bytes/参数（权重/梯度/动量/方差/可能的 master fp32）；激活额外占大头，受批次与序列长影响。
6. 问：去重与温度采样在数据混合中的作用？
   答：去重降泄漏与过拟合；按质量与多样性混合不同来源，并用温度或分层采样维持长尾知识。
7. 问：DPO / KTO / ORPO 相比 PPO 的优势？
   答：不需要在线采样与价值模型，直接用成对偏好或单偏好样本做“离线偏好优化”，实现更简洁高效的对齐。
8. 问：损失监控与早停如何设置？
   答：跟踪训练/验证 ppl、长度归一化 loss、拒答比、毒性指标；设耐心值与阈值，防止过拟合与崩溃。
9. 问：为何要做“拒答/澄清”训练？
   答：在敏感/未知/缺证据时模型应当安全退让或索取信息，减少误导性回答与合规风险。

## 微调与指令跟随（PEFT）

1. 问：LoRA 的核心思想？
   答：将大矩阵分解为低秩增量 ΔW=BA\Delta W=BA 并只训练 A/B（其余冻结），以极低显存实现高效微调。
2. 问：QLoRA 与 NF4？
   答：把基座权重量化到 4-bit NF4（正态感知），保持 16-bit 计算与 LoRA 适配器训练，大幅减少显存。
3. 问：选择 LoRA 目标层的经验？
   答：常对 Wq,Wk,Wv,WoW_q,W_k,W_v,W_o 与 MLP 投影加 LoRA；r=8–64 视任务而定；rank 太小欠拟合、太大易过拟合/耗显存。
4. 问：指令数据的关键特征？
   答：多样、干净、上下文充足、格式稳定（role/工具调用）、覆盖拒答/澄清/引用来源；难例和边界例尤为重要。
5. 问：多任务 SFT 如何防遗忘？
   答：混合任务分布、重放高权重样本、正则化（如 KL 约束）、参数隔离（Adapter/Prefix）与持续学习策略。

## 推理、部署与量化

1. 问：KV Cache 是什么？
   答：缓存已生成 token 的 K/V，推理每步只算新 token 的注意力，显著提升长序列吞吐。
2. 问：KV Cache 显存估算？
   答：约为：层数×头数×head_dim×序列长×2×bytes。示例：32 层×32 头×128×8k×2×2B ≈ 4.2GB（BF16 则翻倍）。
3. 问：长上下文推理的关键优化？
   答：PagedAttention/vLLM 的分块 KV；FlashAttention-2/SDPA；连续批处理/请求合并；提示缓存；张量并行对齐。
4. 问：吞吐与延迟的权衡如何做？
   答：提升并发批大小与连续批处理增吞吐；减少首 token 延迟靠更快的调度、prompt cache、编译内核与轻量前缀。
5. 问：常见量化路线：GPTQ、AWQ、GGUF 有何差异？
   答：GPTQ 偏权重量化后校正；AWQ 关注激活离群保护关键通道；GGUF 是 llama.cpp 生态通用格式，便于 CPU/移动侧推理。
6. 问：INT8/4 量化的影响与补救？
   答：可能损失精度、增误差累积；可做分层敏感度、混合精度、量化感知训练（QAT）或只量化部分层。
7. 问：服务框架如何选：TGI、vLLM、TensorRT-LLM？
   答：TGI 上手容易、生态全；vLLM 强调 PagedAttention 与高吞吐；TensorRT-LLM 靠编译优化/FP8 获得极致性能（NVIDIA GPU）。
8. 问：多卡推理如何切分？
   答：张量并行（分裂矩阵维度）、流水线并行（分层）、张量并行+KV 分片；需要与调度/缓存策略配合。

## RAG 与向量库

1. 问：RAG 的最小闭环？
   答：检索（向量/稀疏/混合）→ 重排 → 构造上下文（聚合/去冗）→ 生成；保证可追溯与引用来源。
2. 问：向量召回与重排如何配合？
   答：先用稠密检索（如 bge/eba-emb）快速召回，再用交叉编码器（cross-encoder）重排，提高精确度。
3. 问：Chunking 的经验参数？
   答：语义块 200–800 tokens，重叠 10–20%；按文档结构分块优于盲目切分；对表格/代码用结构化抽取。
4. 问：避免“检索却不引用”的技巧？
   答：在提示中显式要求“仅依据给定片段回答并标注引用”，并对越权回答设置拒答模板与惩罚。

## 评测与安全

1. 问：Perplexity（困惑度）与下游指标的关系？
   答：PPL 评估语言建模质量，但与任务指标（MMLU、Code、QA）不完全一致；需多维评测（准确率、事实性、毒性等）。
2. 问：常见基准有哪些？
   答：通用（MMLU/CMMLU/CEVAL/IFEVAL）、代码（HumanEval/MBPP）、检索问答（HotpotQA）、多模态（MMMU/MathVista）等。
3. 问：如何做 A/B 与线上评测？
   答：离线基准＋小流量灰度＋用户反馈闭环；监控拒答率、攻击通过率、事实引用率、延迟/吞吐、崩溃率。
4. 问：安全与对抗（越狱）防护？
   答：拒答策略、分级内容过滤、工具权限最小化、提示防御（分层模板/沙箱）、红队集/攻击样本联合训练。

## 多模态、音频与代理

1. 问：多模态 LLM 的典型结构？
   答：视觉编码器（ViT/CLIP/ConvNeXt）→ 投影到 LLM 嵌入空间 → 文本解码器；训练用图文对齐、指令数据与视觉指令微调。
2. 问：工具调用/代理（Agents）如何落地？
   答：在提示或系统层定义函数模式与约束，模型根据函数签名生成调用参数；执行结果回填后继续推理（ReAct/Plan-Execute），并加入记忆与安全审计。
